## 4) Will’s Major Engineering Updates (Walkthrough Summary)

### Refactor (MVVM + Services + DI)
- Reorganized architecture into **MVVM** + **Services**, with clearer ownership:
  - Views: UI only  
  - ViewModels: view state + actions  
  - Models: data structs  
  - Services: functional components created/owned by `SessionModel`
- Implemented **dependency injection** (services are created in order; dependencies passed via constructors).
- Removed **research consent** gating screen as part of refactor.
- Fixed a bug where sessions could start/stop multiple times.

### Experiment Configuration (randomized per-session)
- Added `ExperimentModel` + configurable UI (gear icon) to set **min/max ranges**.
- On session start, the app **randomizes** parameters within ranges:
  - Context window sizes, delays, evaluator selection, etc.
  - Supports a **large LLM** and a **mini LLM** for different workloads.
- Config persists via **UserDefaults** with reset capability.

### Data Export (local)
- Added export window listing session folders; user can select sessions and **export as a zip** via share sheet (AirDrop workflow intended).
- Cloud upload via Supabase is **not implemented yet**; Will created a basic Supabase setup but is still deciding:
  - Store raw files vs structured DB records
  - Whether experiment params should become **table columns** vs remain JSON

### Concurrency / stability improvements
- Standardized event patterns:
  - Combine `PassthroughSubject` for multi-subscriber events (e.g., transcript chunks)
  - Single callback for single-consumer events (e.g., trigger -> prompt)
- Used **actors** to prevent data races (e.g., artifact logging, transcript queues).
- Corrected threading approach:
  - Avoid freezing UI by using `Task.detached` for background work
  - Use **weak self** capture patterns to prevent retention issues

### Future enhancement
- Will wants to display a **session number** during/after a session so Farrokhi can link paper notes to collected artifacts (planned for next sprint).
